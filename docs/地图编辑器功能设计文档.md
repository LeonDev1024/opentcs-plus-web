## 1. 产品概述

### 1.1 产品定位

OpenTCS Web 地图编辑器是基于 OpenTCS（开放运输控制系统）的 AGV 调度管理系统核心组件，提供基于 Web 的可视化地图编辑能力，支持点位、路径、位置区域等元素的创建、编辑和管理。

### 1.2 核心价值

- **可视化编辑**：通过图形化界面创建和管理 AGV 运行地图
- **实时预览**：编辑结果即时渲染，支持多种视图模式
- **数据互通**：支持与 OpenTCS 核心系统双向数据同步
- **用户友好**：符合工业软件设计规范，操作直观高效

## 2. 核心设计

### 2.1 地图元素职责：点、路径与位置（路网与业务建模分离）

编辑器中的三类主数据与 OpenTCS 类 **plant model** 分层一致：**拓扑/可行驶路网** 与 **业务位置** 分开建模，避免「全用点凑合」导致语义混淆。

| 元素 | 主要回答的问题 | 典型用途 | 几何形态 |
|------|----------------|----------|----------|
| **点（Point）** | 路网上**拓扑节点**在哪里 | 路径端点、分叉、停靠拓扑 | 平面上的点（含类型：停车点、充电点等） |
| **路径（Path）** | 车**如何连接**两点、可否双向、代价与形状 | 行驶边、方向、贝塞尔/折线几何 | **线**（及控制点） |
| **位置（Location）** | 业务上**哪儿**装卸/停车/区域规则 | 订单目的地、工位语义、位置类型与属性；可与路网通过 **Link** 等衔接 | **点或面**（矩形中心点、规则区域多边形等） |

**设计原则：**

- **路（Path）≠ 位置（Location）**：Path 描述**可行驶连接**；Location 描述**业务语义锚点**（工位、库位、禁区/限速区等）。保留 Location 更符合仓储/制造业务语言与后端订单模型。
- **点与位置的分工**：Point 侧重**图论顶点**与路径编辑；Location 侧重**对业务系统可见的「地方」**。实践中常见 **Location 经 Link 挂接到某个 Point**，既保证调度可走图，又保证上游系统仍用「位置」下单与统计。
- **Link 路径**：表示位置与路网点之间的**衔接边**（编辑器中常与图层组 `Links` 或名称前缀 `Link` 对应），与「普通行驶路径」同属 Path 数据形态，但**语义**上服务于位置—拓扑连接，不宜与「纯路网边」混为一谈。

后续交互与图层顺序（见下文）在实现层支撑上述分工，而非替代数据模型。

### 2.2 Konva 画布：Stage 内 Layer 顺序

地图主画布在 `MapCanvas.vue` 中基于 **vue-konva / Konva Stage** 组织多个 `Layer`。Konva 按 **Stage 子节点顺序** 从下往上绘制，**后声明的 Layer 在更上层**，且 **鼠标命中检测优先命中最上层可交互图形**。

#### 2.2.1 Layer 顺序（自下而上）

| 顺序 | Layer（`name`） | 说明 |
|------|-----------------|------|
| 1 | `raster-background` | 栅格/导航底图；`listening: false`，不参与交互 |
| 2 | `stage-area` | 全画布透明矩形，用于在空白处触发框选、添加点等（mousedown） |
| 3 | `path` | 路径层：连线、方向箭头、选中路径时的贝塞尔控制点 |
| 4 | `location` | 位置层：业务位置矩形/规则区域、图标、标签、拖拽透明层等 |
| 5 | `point` | **点位层置于路径与位置之上**，便于画线后仍易选中点 |
| 6 | `temp` | 临时层：路径/虚线预览、框选矩形、仿真车辆、问题标记等，**始终在最上** |

#### 2.2.2 设计要点

- **点位置顶**：若点位画在路径层之下，路径线段会盖住点，导致点难以点选；将 `point` 放在 `path`、`location` 之后，保证编辑流（先布线再调点）下的可操作性。
- **路径层内部绘制顺序**（同一 Layer 内）：先绘制**全部路径线段**，再绘制**全部箭头**，最后绘制**选中路径的控制点**，避免同一边上存在 A→B 与 B→A 两条线时，后画的整条线盖住另一条路径的箭头。
- **临时层最上**：预览与操作辅助（框选、仿真车等）需覆盖静态元素，故 `temp` 置于最后。

#### 2.2.3 实现位置

- 代码：`src/views/opentcs/map/editor/components/MapCanvas.vue` 中 `<v-stage>` 下各 `<v-layer>` 声明顺序。

#### 2.2.3.1 模型原点坐标轴（X/Y、O(0,0)）

- **与地图同层平移**：原点装饰线不再使用外层面板上的固定百分比定位；在 **`map-canvas-container` 内**用 **`canvasState.offsetX` / `offsetY`**（与 Konva Stage 的平移一致）定位，使模型 **(0,0)** 与 Stage 上该点重合；**漫游拖拽**改变 offset 时，坐标轴与栅格/地图内容**一起移动**。
- **默认视口**：当 **`offsetX`/`offsetY` 均为 0**（新会话、重置视图等）且容器尺寸已就绪时，自动设为 **左下角附近**（距左、下各约 48px），使模型原点 **(0,0)** 出现在画布**左下区域**；若地图数据里已保存非零 offset，则**不覆盖**。
- **两套坐标指示（重合时实线优先）**：
  - **工厂/场景原点 O(0,0)**：使用实线（蓝 X + 红 Y），始终可见且不参与编辑。
  - **地图原点 M(x,y)**：使用虚线（同配色），由 `mapData.mapInfo.originX/originY` 决定；当 M 与 O 重合时，视觉上以实线为主，避免“双重十字”影响阅读。
- **地图原点编辑交互**：在属性面板 `Layout` 区域提供开关（拖拽编辑）。开启后，在虚线原点 **M** 旁显示橙色圆形拖动手柄；拖动时更新 `mapData.mapInfo.originX/originY`，同时保持工厂原点 O(0,0) 不变。坐标轴线/标签默认不参与命中（`pointer-events: none`），仅在编辑模式下对手柄开启交互，避免影响点位/路径/位置的绘制与选择。

#### 2.2.4 路径工具（连线）交互

- 在「连线」工具下：**先点击起点**，移动鼠标时 **临时层仅显示预览连线（无方向箭头）**。
- **在终点点击** 完成创建，生成 Path 并绘制 **方向箭头**；**行驶方向由点击顺序唯一确定**（先点 = 起点，后点 = 终点），**不在属性面板里再选手动方向**。
- **不在 mouseup 时** 完成连线或清除预览（避免先于 `click` 清空起点、或出现悬浮箭头）；取消：点击空白处或切换工具。
- **路径视觉（带状路网）**：预览与正式路径**同线宽**；预览为**半透明实线带状**（不用粗虚线——Konva 圆端帽会在虚线段端叠成圆斑）；预览绘制在**路径层**、位于点位层之下，避免盖住点标签；正式路径为天蓝半透明实线；方向箭头随线宽放大，浅色描边时用深色箭头填充。
- **点与路径尺度**：默认 **点外接圆直径 ≈ 路径带状线宽 × (1 + 1/3)**（同级比较：点比路径宽约 1/3）；类型间差异在 `POINT_TYPE_RADIUS` 中按比例保留。常量：`src/utils/mapEditor/mapVisualTokens.ts`（`PATH_RIBBON_STROKE_WIDTH` 与 `DEFAULT_POINT_OUTER_RADIUS` 联动）。
- **虚线链接（位置↔点）**：使用 **`DASHED_LINK_STROKE_WIDTH`**（约 5px）与 **`DASHED_LINK_DASH_PATTERN`**，与带状行驶路径的默认线宽**分开计算**，避免虚线被误用带状宽度（18px）显得过粗。

#### 2.2.5 地图管理控制台：地图原点编辑（预览区）

- **入口位置**：在地图管理控制台 `src/views/opentcs/map/index.vue` 的预览区右上角，`新建地图`、`地图编辑` 旁新增 **原点编辑** 按钮（需先选中某张地图）。
- **编辑模式**：点击 **原点编辑** 后显示浮层（X/Y 输入框 + 保存/取消）。同时预览区展示虚线坐标轴（地图原点 **M(x,y)**），并在虚线原点附近显示橙色拖拽手柄。
- **重合规则**：当地图原点 **M** 与工厂/场景原点 **O** 完全重合（等价于 `originX=0 && originY=0`）时，为避免视觉重复，预览区仅保留实线 O(0,0)；虚线 M(x,y) 隐藏。
- **拖拽与输入联动**：
  - 拖拽手柄会实时更新浮层中的 `originX/originY`（单位：mm）。
  - 直接修改输入框也会驱动虚线坐标轴位置同步刷新。
- **持久化**：点击保存后通过 `updateNavigationMap` 将 `originX/originY` 写回后端，并同步更新本地地图列表数据；取消则回滚到进入编辑模式前的备份值。
